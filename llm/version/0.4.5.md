# v0.4.5 - ReplyBuilder の ephemeral バグ修正と editReply() 追加

## 概要

`followUp()` に `ephemeral` フラグが渡されていないバグを修正し、`editReply()` メソッドを追加した。

## 変更内容

### 修正
- `src/interaction/reply.ts` - `followUp()` のペイロードに `ephemeral: this.is_ephemeral` を追加

### 追加
- `src/interaction/reply.ts` - `editReply()` メソッドを追加。`deferReply()` 後に元のメッセージを上書きする用途。ephemeral は `deferReply()` 時の設定が引き継がれるためペイロードには含まない
- `src/messages/errors.ts` - `REPLY_EDIT_FAILED` エラーコードを追加

### 変更なし（破壊的変更なし）

## Discord 仕様との対応

| メソッド | ephemeral の扱い |
|---|---|
| `send()` | ペイロードに `ephemeral` を含める |
| `defer()` | `deferReply({ ephemeral })` に渡す |
| `followUp()` | ペイロードに `ephemeral` を含める（今回修正） |
| `editReply()` | `deferReply()` の ephemeral が引き継がれるため不要 |
