# v0.4.0 - voiceモジュールの分離エントリーポイント対応

## 概要

`@discordjs/voice` / `@discordjs/opus` / `prism-media` が起動時に常にロードされる問題を解消。
voice機能を `@philtzjp/atac/voice` の別エントリーポイントに分離し、使用しないBotで不要なネイティブモジュールがロードされないようにした。

## 変更内容

### 追加
- `src/voice/index.ts` - voiceモジュール専用エントリーポイント（`VoiceRecorder`クラスとvoice関連型をre-export）
- `package.json` exports `"./voice"` エントリー（`./dist/voice/index.js` / `./dist/voice/index.d.ts`）
- `package.json` `peerDependenciesMeta` - `@discordjs/opus`、`@discordjs/voice`、`prism-media`、`better-sqlite3`、`neo4j-driver` をoptionalとしてマーク

### 変更
- `src/index.ts` - `VoiceRecorder`クラスのエクスポートおよびvoice関連型のエクスポートを削除
- `package.json` - `@discordjs/opus`、`@discordjs/voice`、`prism-media` を `dependencies` から `peerDependencies` に移動
- `package.json` - バージョン `0.4.0` へ更新

### 破壊的変更
- `import { VoiceRecorder } from "@philtzjp/atac"` が廃止
  - **移行先**: `import { VoiceRecorder } from "@philtzjp/atac/voice"`
- voice関連型（`VoiceRecorderConfig`、`RecordingSession`、`RecordingSegment`、`ParticipantInfo`）のインポートパスも同様に変更
  - **移行先**: `import type { VoiceRecorderConfig } from "@philtzjp/atac/voice"`
