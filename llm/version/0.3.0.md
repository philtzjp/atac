# v0.3.0 - データベース・HTTP・Docker対応

## 概要

Neo4j / SQLite3 アダプター、HTTPクライアント、Docker対応を追加。

## 変更内容

### 追加
- `src/database/types.ts` - DB関連型定義（Neo4jConfig, SQLiteConfig, Neo4jTransaction）
- `src/database/neo4j.ts` - Neo4jClient（connect, disconnect, query, queryOne, transaction）
- `src/database/sqlite.ts` - SQLiteClient（connect, disconnect, query, queryOne, execute, transaction）
- `src/http/types.ts` - HTTP関連型定義（HttpClientConfig, RequestOptions, HttpResponse）
- `src/http/client.ts` - HttpClient（get, post, put, patch, delete）
- `Dockerfile` - マルチステージビルド（node:18-alpine）
- `docker-compose.yml` - Bot + Neo4j 開発環境
- `.dockerignore`

### 変更
- `src/index.ts` - 新モジュールのexport追加
- `src/messages/errors.ts` - DB・HTTPエラーコード11個追加
- `src/messages/logger.ts` - DB・HTTPログメッセージ8個追加
- `package.json` - peerDependencies に neo4j-driver, better-sqlite3 追加、バージョン 0.3.0 へ更新
- `.env.example` - Neo4j環境変数追加

### 新規エラーコード
- NEO4J_CONNECTION_FAILED, NEO4J_QUERY_FAILED, NEO4J_TRANSACTION_FAILED, NEO4J_NOT_CONNECTED
- SQLITE_CONNECTION_FAILED, SQLITE_QUERY_FAILED, SQLITE_TRANSACTION_FAILED, SQLITE_NOT_CONNECTED
- HTTP_REQUEST_FAILED, HTTP_TIMEOUT, HTTP_INVALID_RESPONSE

### 新規ログメッセージ
- NEO4J_CONNECTED, NEO4J_DISCONNECTED, NEO4J_QUERY_EXECUTED
- SQLITE_CONNECTED, SQLITE_DISCONNECTED, SQLITE_QUERY_EXECUTED
- HTTP_REQUEST_SENT, HTTP_RESPONSE_RECEIVED

### 依存関係変更
- `neo4j-driver` を peerDependencies に追加
- `better-sqlite3` を peerDependencies に追加
- `@types/better-sqlite3` を devDependencies に追加
