# v1.0.0 - ドキュメント・エージェントスキルパッケージへの移行

## 概要

ATAC Coreを npm TypeScriptライブラリ（`@philtzjp/atac`）から、LLMエージェント向けの設計仕様書＋エージェントスキルパッケージに完全移行。LLMに毎回最新の指示を渡して実装を生成させるアプローチに転換した。

## 破壊的変更

### 削除

- `src/` — 全TypeScriptソースコード（24ファイル）
- `dist/` — コンパイル済み出力
- `package.json` / `package-lock.json` — npmパッケージ定義
- `tsconfig.json` — TypeScript設定
- `.eslintrc.json` / `.prettierrc` — コードフォーマット設定
- `.npmrc` — npmレジストリ設定
- `Dockerfile` / `docker-compose.yml` / `.dockerignore` — Docker設定
- `.github/workflows/ci.yml` / `publish.yml` — CI/CDパイプライン
- `.env.example` — 環境変数テンプレート
- `config/` / `temp/` — 空ディレクトリ
- `node_modules/` — 依存パッケージ

### 変更

- `CLAUDE.md` — TypeScript開発規約からLLMエージェント向け指示書に改定
- `llm/ARCHITECTURE.md` — フレームワーク説明から、src/の全実装知識を含む実装ガイドに改定
- `.gitignore` — 簡素化

### 追加

- `.agents/skills/chat-sdk/SKILL.md` — Chat SDKエージェントスキル
- `.claude/skills/chat-sdk` — Claude Codeスキルシンボリックリンク
- `skills-lock.json` — スキルインストーラロックファイル
- `llm/version/1.0.0.md` — 本ファイル

### 維持（変更なし）

- `llm/models.yaml` — データモデル定義
- `llm/version/0.2.0.md` ~ `0.4.5.md` — 過去バージョン履歴
- `LICENSE` — MPL-2.0

## 移行ガイド

`@philtzjp/atac` を `import` して使用していたプロジェクトは、以下の手順で移行：

1. `npm uninstall @philtzjp/atac`
2. プロジェクトの `CLAUDE.md` にATAC Coreの規約を組み込む
3. `llm/ARCHITECTURE.md` と `llm/models.yaml` をLLMエージェントのコンテキストとして渡す
4. LLMエージェントに同等の実装を生成させる
